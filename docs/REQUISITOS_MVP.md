
REQUISITOS FUNCIONAIS 

DESCRIÇÃO 

Cadastro de Usuário 

O sistema deve permitir que o usuário crie uma conta informando nome, email, telefone e endereço. 

Login de Usuário 

O sistema deve permitir que o usuário acesse sua conta com email e senha. 

Visualizar Catálogo de Flores 

O sistema deve exibir a lista de flores e arranjos disponíveis, com imagem, descrição e preço. 

Visualizar Detalhes do Produto 

O sistema deve mostrar informações detalhadas de um arranjo selecionado. 

Adicionar ao Carrinho 

O usuário deve poder adicionar um item ao carrinho de compras. 

Remover Item do Carrinho 

O sistema deve permitir excluir produtos do carrinho. 

Finalizar Pedido 

O usuário deve conseguir realizar um pedido informando forma de entrega e dados de confirmação. 

Agendar Entrega 

O sistema deve permitir escolher data e período de entrega (manhã/tarde/noite). 

Acompanhar Status do Pedido 

O sistema deve exibir status do pedido: Em preparação, Saiu para entrega, Entregue. 

Pesquisa de Produtos 

O sistema deve permitir pesquisar flores ou arranjos por nome ou categoria. 

 

REQUISITOS NÃO FUNCIONAIS 

DESCRIÇÃO 

Usabilidade 

A interface deve ser simples, clara e adaptada para usuários de qualquer idade. 

Responsividade 

O aplicativo deve funcionar corretamente em diferentes tamanhos de tela. 

Disponibilidade 

O sistema deve estar disponível 24 horas por dia. 

Desempenho 

O aplicativo deve carregar o catálogo em no máximo 3 segundos em internet comum. 

Segurança 

Os dados dos usuários devem ser armazenados de forma segura (criptografia de senha).

Adicionar arranjos ao carrinho 

Confirmar compra (simulada) 

Agendar entrega 

Escolha de data e período (manhã/tarde/noite) 

Cadastrar usuário e salvar informações básicas 

Nome, telefone e endereço de entrega 

Validar aceitação do público ao modelo digital da floricultura 

Coletar feedback sobre usabilidade, agilidade e variedade
